Homepage architecture is split into three boundaries: `src/content` provides locale-specific page data, `src/sections` owns section-level composition and markup for Section01-Section09, and `src/components/ui` contains reusable presentation primitives (`PageFrame`, `SectionShell`) shared by sections; cross-cutting helpers live in `src/lib` (`i18n`, `media`, `animation`) so section components stay focused on structure and consume typed content only, which keeps routing/content logic isolated and supports DRY/KISS as features grow.
